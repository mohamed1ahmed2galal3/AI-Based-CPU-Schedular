Section 2 — Problem Statement

Traditional schedulers
treat everyone the same

Two completely different process types — one treatment. That's the problem.

⚙
CPU-bound
  • Needs the CPU constantly
  • Long burst times (40–100ms)
  • Examples: video encoding, ML training, calculations
  • Benefits from large time quantum
⟳
I/O-bound
  • Mostly waiting for disk / network
  • Short burst times (2–25ms)
  • Examples: database queries, file I/O, browser
  • Benefits from small quantum + high priority

FCFS suffers from the convoy effect — slow processes block fast ones.
Round-Robin gives equal quantum to all — no intelligence, wasted context switches.

Section 8.6 — Limitations

Honest about our limitations

!
Current limitations
  • Synthetic dataset — real OS processes are more complex
  • No context-switch overhead modeled in simulation
  • Cold-start issue for unseen process types
  • Fixed quantum per type — not adaptive to system load
→
Future improvements
  • Test on real OS process trace data
  • Add context-switch cost to simulation
  • Implement dynamic quantum based on real-time load
  • Deploy feedback loop for continuous retraining
